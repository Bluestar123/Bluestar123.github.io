发表于 2017-12-21 | 分类于 python学习

Django开发常见错误

1.Application labels aren’t unique
解决：查看settings.py中INSTALLED_APPS内容是否重复。

2.error: Microsoft Visual C++ 9.0 is required (Unable to find vcvarsall.bat). Get it from http://aka.ms/vcpython27

解决：下载安装后执行VCForPython27.msi。

3.error: command “C:\Users\qwf\AppData\Local\Programs\Common\Microsoft\Visual C++ for Python\9.0\VC\Bin\cl.exe”

解决：在http://www.lfd.uci.edu/~gohlke/pythonlibs/#mysql-python下载对应的包版本，如果是win7 64位2.7版本的python，就下载安装：MySQL_python-1.2.5-cp27-none-win_amd64.whl
4.ProgrammingError at /admin/login/(1146, “Table ‘blog_projectdb.auth_user’ doesn’t exist”)


5.view must be a callable or a list/tuple in the case of include().
django 1.10之后不在支持URL用字符串表示了

6.auth.User.groups: (fields.E304) Reverse accessor for ‘User.groups’ clashes with reverse accessor for ‘User.groups’
解决：在settings.py中配置AUTH_USER_MODEL = ‘applicationName.User’

7.django.db.migrations.exceptions.InconsistentMigrationHistory: Migration admin.0001_initial is applied before its dependency users.0001_initial on database ‘default’
解决：

删除数据库中 除了auth_user的其他表，然后重新来一次

大概原因是因为admin的模型依赖了之前默认的user模型吧

8.ImportError: cannot import name RadioFieldRenderer
django版本的问题，换成1.9就可以了

9.Forbidden (CSRF cookie not set.): /HomeMe/php/form-process.php
CSRF问题,查看：http://www.cnblogs.com/jasonhaven/p/7354994.html

10.关于发邮件报错535 Error：authentication failed解决方法
设置settings.py,注意密码使用：客户端授权码

11.SMTPSendFailedException: 553 Mail from must equal authorized user
做邮件发送时，登录用户和发信人必须一致；而在测试时我登录用户的邮箱是163邮箱，发信人邮箱是QQ邮箱，导致报错
