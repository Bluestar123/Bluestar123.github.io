python正则表达式

发表于 2017-12-17 | 分类于 python学习

Python 的规范库里确实有库，re 库属于一致很流行的库。虽然它真的有年头没有更新了，但是在我看来，仍不失爲静态言语中最好的库之一。

我觉得 Python 作爲一种静态言语，居然没有对正则表达式停止原生支持，真是少见。虽然没有提供（原生的）语法和解释器的支持，但（这个模块）从纯 API 的角度给出了一个设计愈加完善的中心零碎作爲补充的处理方案。但是这个方案也挺诡异的，比如说，它的解析器是用纯 Python 写的，假如你导入库的同时去追踪 Python 就会发生一些很诡异的后果。最初你会发现本人90%的工夫都花在了 re 的支持库上。

久经考验

经过工夫的沉淀，正则库早已成爲历代 Python 规范库中不可或缺的局部。 Python3 就另当别论了，我觉得除了添加了对 unicode 的支持以外，它从始至今没什麼实质的提升。到如今，成员枚举都是乌七八糟的（不信就去试试看，对一个正则对象用 dir() 函数能前往什麼东西）。

用了这个正则库最大的益处就是十分波动，任它 Python 版本更替，我自巍然不动。你的 Python 曾经不是当年的 Python了，你的 re 永远是你的 re。思索到我写过那许许多多的正则表达式，却历来没有由于 re 库的变化而重写，那必需是满满的幸福啊。

这个库有一点我觉得设计的挺神奇的，它的结构（compiler）和解析（parser）函数是用 Python 写的，但是匹配（matcher） 函数是用 C 写的。这意味着假如我们情愿的话，就可以将解析器的外部构造传递给编译器，从而完全绕过正则表达式的解析。虽然文档里没写，但现实上的确可以这麼干。

还有很多这种例子，但是在（官方）文档中的正则局部都没有收录，或许没讲清楚，所以上面我就给大家演示几个例子，让你见识见识 Python 的正则库究竟有多炫酷。

空既是色

想在 Python 中运用正则表达式完成逆匹配（一个 pattern 与指定字符串不匹配）普通来说比拟费事。假定我们要写一个相似维基言语那样（比方 Markdown 言语）的语法剖析器。除了那些表示特定格式的语法标志，两头还有许多文本也需求我们来处置。这时我们只想匹配那些已知的标志符号，但是两头还有很多别的内容（非语法标志）也需求处置。怎样才干跳过这些内容呢？

一种方案是编译一组正则表达式然后放到一个列表里，逐一去尝试匹配。假如全部匹配失败，就跳过以后字符（然后持续匹配）。

这个方案既不优雅也不高效。普通来说，匹配失败的状况越多代码效率就越低。由于那样就意味着我们每次只能向后跳过一个字符，而且还是用的 Python 这种解释型的言语（来循环）。同时，这种方案灵敏性也不够好，每次只能匹配到对应的标志符号，假如还要匹配分组就只能再把这段重新扩展一下。

难道就没什麼好方法了吗？我们就不能让正则引擎间接去扫描指定的一批正则表达式吗？

上面有意思的来了。实践上，假如我们把表达式写成 （a | b）这种分枝条件的款式，它就会同时搜索能否匹配 a 或许 b。所以我们可以把要匹配的一切语法标志全部这样写到一同，然后去匹配就好了。这麼匹配写起来很方便，但是匹配后果你一定一脸懵比，由于完全不晓得是被那一堆表达式中的哪一个匹配成功的。
