hexo搭建博客并部署到github上

发表于 2017-12-05 | 分类于 搭建博客

#hexo搭建博客并部署到github上
大家也搭建过博客，很多时候，按着教程来做就可以了，但是我当时为了搭建Hexo博客并部署到Github，走了不少弯路。现在终于搭建出来了，为了帮助大家，我决定写一篇“史上最详细“截图”搭建Hexo博客并部署到Github”。

工具

  • windows
  • git
  • node.js

安装hexo

1.利用npm命令安装，在任意位置点击右键，选择git Bash。

2.输入命令：

npm install -g hexo
注意：-g是指全局安装

创建hexo文件夹

安装完成后，在你喜爱的文件夹下，执行以下命令（选择git bash），hexo即会自动在目标文件夹建立网站所需要的所有文件。
hexo init

本地查看

现在我们已经搭建起本地的hexo博客了，执行以下命令，然后在浏览器输入localhost：4000
hexo generate
hexo server

注册github账号

  • 这个就很简单了

创建Repository
创建的时候注意Repository的名字。比如我的Github账号为Bluestar123，那么我应该创建的名字是Bluestar123.github.io

修改配置文件

  • 到你刚刚创建爱你的Repository下，找到以下内容

  • 先点击HTTPS，然后复制里面的地址。然后编辑_config.yml文件

  • 修改文件里面的deploy，其中的repository就改成你刚刚复制的地址，保存这个文件。

设置SSH keys

  • 在git bash输入以下命令，检查是否已经存在了SSH keys

ls -al ~/.设置SSH

  • 如果不存在就没有关系，如果存在的话，直接删除.ssh文件夹里面所有文件。
  • 输入以下指令（邮箱就是注册github的邮箱）后，回车
    ssh-keygen -t rsa -C “704465184@qq.com”
  • 然后它会提示要你输入passphrase（如上图，没有输入的话直接回车，如果你输入的话，要记得，到时候会用到），之后，如果出现类似下图：

  • 然后键入以下指令
    ssh-agent -s

  • 继续输入指令：
    ssh-add ~/.ssh/id_rsa

  • 输入之后，在我这里时出错了，不知道你的有没有

  • 如果你也是这样的话，就输入以下指令：
    evalssh-agent -s
    ssh-add

到了这一步，酒可以添加SSH key到你的github账户了，键入一下指令，拷贝key
clip < ~/.ssh/id_rsa.pub

  • 然后到github里面，点击右上角的设置图标

  • 在settings sidebar里面，点击SSH keys：

  • 点击add SSH key

  • 输入title，作为这个key的描述（你可以输入personal macbook air，瞬间高大上）

  • 然后这个key就是刚刚拷贝的，你直接粘贴就好（也可以文本打开以下文件）
  • 点击Add Key
  • 输入你的github密码即可完成SSH key的添加。恩，最后还是测试一下吧，键入以下命令：
    ssh- T git@github.com
  • 你可能会看到有警告，没事，输入‘yes’就好

完成部署

  • 最后一步，快要成功了，键入指令
    hexo generate
    hexo deploy

  • OK,我们的博客就已经完全搭建起来了，在浏览器输入（当然，是你的用户名）：
    我的博客

注意：每次修改本地文件后，需要键入hexo generate才能保存。上传到github上面时，就应该键入hexo generate之后,在键入hexo deploy，大概成功之后是这样：

  • 对了，记住上图的username是你的github账号名，不是邮箱，password是你的github密码。

hexo现在支持更加简单的命令格式了：

hexo g == hexo generate
hexo d == hexo deploy
hexo s == hexo server
hexo n == hexo new

可能遇到的问题

ERROR Deployer not found: git 或者 ERROR Deployer not found: github

解决方法： npm install hexo-deployer-git –save

如发生报错： ERROR Process failed: layout/.DS_Store , 那么进入主题里面layout和_partial目录下，使用删除命令：

rm-rf.DS_Store

ERROR Plugin load failed: hexo-server

原因：

Besides,utilities are separated into a standalone module.hexo.util is not reachable anymore.

解决方法，执行命令：

sudo npm install hexo-server

执行命令hexo server，提示：Usage: hexo ….

原因：

我认为是没有生成本地服务

解决方法，执行命令：

npm install hexo-server –save

提示：hexo-server@0.1.2 node_modules/hexo-server
